# Prompts Guide

이 문서는 `prompts/` 폴더에 AI 프롬프트와 그 결과물을 기록하는 최소 가이드입니다.

## 저장 기준
- 코드 변경이 실제로 반영된 경우
- 버그 수정·리팩토링·성능 개선 등 문제 해결에 기여한 경우
- 설계·아키텍처 결정, 중요한 인사이트가 도출된 경우

## 디렉토리 구조
- 경로: `prompts/{주차}/{기능 또는 주제}.md`
- 예시: `prompts/1주차/로그인_API_구현.md`

## 파일 명명 규칙
- `snake_case` 권장
- 내용이 무엇인지 바로 알 수 있도록 핵심 주제 명시
- 예시: `로그인_API_구현.md`

## 📌 문서 템플릿
각 프롬프트 파일은 아래 형식을 따릅니다.

```markdown
# {주제/기능 이름}

## 1. 배경 및 목적

- 어떤 작업을 수행하려고 했는지 간단히 기술

## 2. 프롬프트 (User Input)

\`\`\`text
(여기에 실제로 사용한 프롬프트 내용을 붙여넣으세요)
\`\`\`

## 3. AI 응답 요약 (AI Output)

- AI가 제안한 핵심 내용 및 코드 스니펫 요약

## 4. 결과 및 적용 (Result)

- 실제 프로젝트에 어떻게 적용했는지, 추가 수정사항
- 테스트/배포 결과나 영향을 받은 부분 기재
```
